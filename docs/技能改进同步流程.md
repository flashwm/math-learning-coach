# 技能改进同步流程

## 目标

在数学讲题会话里，用户提出长期改进意见时，既要立刻改善当前题的讲解，又要把可复用规则同步到 `math-learning-coach` 技能和 `数技` 项目。

## 触发条件

只有用户明确表达长期意图时才同步技能，例如：

- 以后都这样
- 这个要改进技能
- 记到数学技能里
- 同步到数技项目
- 以后讲这类题都按这个规则

如果只是要求“这题重讲”“图重新画”“换个说法”，只修正当前题，不自动改技能。

## 执行动作

1. 先按新要求立刻修正当前题的讲解、图或网页讲义。
2. 把可复用规则写入正式技能目录：

```text
C:\Users\xiao\.codex\skills\math-learning-coach
```

3. 把同一规则同步到维护项目：

```text
E:\CodeX项\数技
```

4. 在 `CHANGELOG.md` 记录改动。
5. 最后告诉用户哪些文件已经更新，是否完成同步。

## 权限限制

如果当前会话没有权限写入正式技能目录或 `数技` 项目，应先继续完成当前题讲解，再提示用户需要打开对应项目或授权文件写入。

## 收录原则

- 写入技能的必须是通用规则。
- 不收录只对某一道题有效的临时细节。
- 不收录未经验证的结论。
- 不把过长案例塞进 `SKILL.md`，案例应放入 `examples`。
