# math-learning-coach 使用与维护说明

这份说明用于快速了解：这个数学技能能干什么、平时怎么使用、发现问题后怎么改进、后期怎么查看维护记录。

## 一句话说明

`math-learning-coach` 是一个面向数学学习的讲解技能，重点不是只给答案，而是把题目讲成学生能理解的学习材料，尤其适合拍照题、初中数学、几何题、函数题和网页讲义。

## 它能干什么

### 1. 讲数学题

适合这类需求：

- 拍照上传一道数学题，让它识别题目并讲解。
- 要求先给结论，再讲思路，再列计算。
- 要求限制在初中、人教版、课堂可接受的方法内。
- 要求不用向量、点积、投影等超纲说法。

常用说法：

```text
用 math-learning-coach 讲这道题，按初中能听懂的方式，先讲思路，再讲计算。
```

### 2. 出网页讲义

适合这类需求：

- 把一道题整理成网页讲义。
- 每一步配图说明。
- 结论、思路、计算、易错点分区展示。
- 几何图、函数图、信息图要求清楚、不重叠、不乱标。

常用说法：

```text
把这道题做成网页讲义，每一步都配图，适合初中学生复习。
```

### 3. 做几何图和函数图

技能要求几何图尽量用确定性的 SVG 或坐标绘图，不依赖随意生成的图片。

重点规则：

- 几何图先验证条件，再画图。
- 折叠题先找对应点和折痕关系。
- 函数图要按真实方程绘制。
- 点、线、标签不能乱放或互相遮挡。

### 4. 沉淀学习方法

它还可以把一道题总结成：

- 这题怎么入手。
- 关键条件是什么。
- 容易错在哪里。
- 同类题下次先检查什么。
- 这类题的通用解题套路。

## 项目和正式技能的关系

这里有两个位置，作用不同。

### 维护项目

```text
E:\CodeX项\数技
```

这是长期维护区，负责保存：

- 技能源码
- 文档规则
- 示例题型
- 改进日志
- 后续想法
- 发布脚本

### Codex 正在使用的技能目录

```text
C:\Users\xiao\.codex\skills\math-learning-coach
```

这是 Codex 实际调用技能的位置。

简单理解：

```text
E:\CodeX项\数技
= 维护仓库、说明书、改进记录

C:\Users\xiao\.codex\skills\math-learning-coach
= Codex 当前真正使用的版本
```

## 平时怎么使用

### 正常讲题

新开一个会话，把题目照片发进去，然后说：

```text
用 math-learning-coach 讲这道题，出网页讲义。
```

如果有年级或教材限制，要直接说清楚：

```text
按初中人教版能接受的方法讲，不要用向量。
```

### 当前题临时调整

如果只是想让当前题讲得更好，可以直接说：

```text
这一步换成更适合初中生的说法。
```

或：

```text
这个图太乱，重新画一版分步骤图。
```

这类反馈只影响当前题，不一定写进长期技能。

### 长期改进技能

如果你希望以后所有类似题都这样处理，要明确说：

```text
这个规则以后都要按这样讲，立刻改进 math-learning-coach，并同步到数技项目。
```

这时应该同时做四件事：

1. 立刻修正当前题讲解。
2. 修改 Codex 正在使用的正式技能目录。
3. 同步回 `E:\CodeX项\数技`。
4. 写入长期改进日志。

## 重要文件怎么看

### `skill-source\math-learning-coach\SKILL.md`

技能最核心的规则文件。

如果希望 Codex 以后默认按某个规则讲题，通常最终要写进这里。

### `docs\`

放更细的长期规范。

例如：

- 初中数学语言边界
- 几何图怎么画
- 网页讲义怎么排版
- 错题怎么归因
- 技能改进怎么同步

### `examples\`

放典型题型，用来测试技能有没有变好。

后面可以继续补：

- 圆综合题
- 动点最值题
- 相似三角形题
- 折叠压轴题

### `logs\skill-improvements.md`

长期连续记录每次技能改进。

以后想查“什么时候为什么改了某条规则”，主要看这个文件。

### `CHANGELOG.md`

记录版本级摘要。

它不需要像日志一样写得很细，只说明这一阶段大概改了什么。

### `IDEAS.md`

放暂时不做、但以后可能有用的想法。

## 常用维护命令

从数技项目发布到 Codex 正式技能目录：

```powershell
cd E:\CodeX项\数技
.\scripts\publish.ps1
```

从 Codex 正式技能目录回同步到数技项目：

```powershell
cd E:\CodeX项\数技
.\scripts\sync-from-installed.ps1
```

## 每次长期改进后要检查

- 当前题是否已经按新规则重新讲好。
- `SKILL.md` 是否只写了通用规则，没有写一题专属细节。
- 相关 `docs` 或 `examples` 是否需要补充。
- `logs\skill-improvements.md` 是否记录了来源、原因、改动文件和同步状态。
- `CHANGELOG.md` 是否补了摘要。
- 项目版本是否已经发布到 Codex 正式技能目录。

## 最推荐的使用习惯

平时学题时不用一直想着维护项目。你只需要在关键时刻说清楚：

```text
这个只是当前题调整。
```

或者：

```text
这个以后都要这样，改进技能并同步到数技项目。
```

这样就能把“正在学习” 和 “长期改进技能”分开，不打断学习，也不会让好经验散在不同会话里。
